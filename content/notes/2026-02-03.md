## Learned today

- I can use arrow function with ts rule strictPropertyInitialization to get enforced member init.
- When using `isolatedModules` rule with typescript i need to export type. Its necessary to avoid exporting types (that might be dropped by a builder like esbuild, wp etc...) since they're
  not used by runtime.

  I.e.:

  ```javascript
  export type Foo = { x: number }
  ```

- When a type is defined like `onmessage: ((this: Worker, ev: MessageEvent) => any) | null;`. Then when init the onmessage, "this" in the function will be the actual "this" in javascript
  The correct implemenation would then add a function to onmessage stating only one parameter, in the above case that would be ev: MessageEvent.
